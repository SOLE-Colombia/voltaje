Share your pocket internet

Welcome

Hi, I’m Gabriel. I’m an artist. Digital devices, how they work and the way we humans interact with them, are interesting to me. I consider myself a tinkerer and I invite you, if you want, to be one too.

I wrote the Raspberry Pi: Una pequeña Internet de bolsillo solution so you could set it up, and now I’ll walk you through how to share your device’s signal with other people or other devices.

What is sharing the internet on your RasperryPi?

Having a computer the size of a credit card, being able to connect it to the Internet and, in addition, being able to share that connection with the other computers you have to do SOLE, is to open the doors to the largest library in the world!

It is to turn any space into a Self-Organized Learning Environment where you can open the doors to the creativity to solve the challenges, complex or simple, of your community… UNBELIEVABLE!

To make SOLE there are only three fundamental things: people, a few computers with internet connection and Big Questions.

What is it for?

With this recipe you can turn your Raspberry Pi into a portable Router, so that your community can connect wirelessly and access the information hosted there, and, if you have a wired Internet connection, you can share it too.

What do you need?

  • A Raspberry Pi
  • A computer
  • Internet connection - you can check the solution ¿Cómo saber qué tipo de señal de Internet tienes? to make sure what kind of connection you have, and if you have it active.
  • A lot of patience

How to do it?

  • Step 1: Connect to your Pi

    • First of all, make sure you have a working Raspberry Pi. If you need help for this, check the recipe **Rasperry Pi: A small pocket Internet** where we tell you how to get started with this device.
    • Once you have your system up and running, connect to your Rasperry Pi via Terminal by initiating an SSH connection.
    • Make sure you have an Internet connection to be able to continue with the following steps.

  • Step 2: Upgrade your Raspberry Pi system

    The first step is to perform an update of the entire [system](../../Conceptorio 12a2bd68c5b6808baf3be5e67cddb488/Conceptorio 1624106bff984c4590342afeae3e5e0d/Sistema informático (SI)%20Information%20system%20(IS)%202382bd68c5b680609579f5e5eb9f0836.md), to ensure that we have the latest versions of the programs running on our Pi.

    To do this, just type in the terminal:

    sudo apt update && sudo apt upgrade

    The system will warn us which packages need to be updated. To continue with the process, we must accept, typing y and pressing the enter key.

  • Moment 3: Install the necessary software

    Next, we must install two programs on our Pi:

    sudo apt install -y hostapd dnsmasq

    This command will install hostapd which is a utility that allows us to use the wireless of our Pi as a wireless access point and dnsmasq which will allow us to configure a DNS and DHCP server to allow users to connect to the server and can have the necessary network configurations to access it.

  • Step 4: Configure a static IP

    • Disable the wireless interface

      In this step we will configure a static IP address on the wireless interface of our PI. It is important that you are not connected wirelessly to the Pi, because during the following steps the wireless communication will be suspended. So, make sure you are connected via a cable, or have a display and keyboard connected directly to the Pi to continue.

      The network settings on the Pi are being handled by the dhpcd program, which is responsible for requesting and assigning the necessary IP addresses for the available network interfaces. At this point, we need to tell this program to ignore the wireless interface. To do this, we have to edit the configuration file:

      sudo nano /etc/dhcpcd.conf

      … and at the end of it, add this line:

      denyinterfaces wlan0

      To exit and save the changes we press ctrl + x followed by y when it asks us.

    • Assign a static IP

      Now we must assign to the wlan0 interface a static IP address in another range different from that of our wired network. Let’s edit the file:

      sudo nano /etc/network/interfaces

      Don’t be afraid. It looks complex, but, if you have come this far, you are not far away. Add the following at the end:

      allow-hotplug wlan0
iface wlan0 inet static
    address 192.168.10.1
    netmask 255.255.255.0
    network 192.168.10.0
    broadcast 192.168.10.255

      What we are doing is configuring this interface to have the IP 192.168.10.1.

      To exit and save the changes, press ctrl + x followed by y when prompted.

  • Moment 5: Configuring our wireless access point

    It is time to tell hostapd how to operate. We will tell it to broadcast the network name(SSID) and to operate on a specific channel. We will put these settings in the file :

    sudo nano /etc/hostapd/hostapd.conf

    … where we add the following data:

    interface=wlan0
driver=nl80211
ssid=SOLEVOLTAJE
hw_mode=g
channel=11
ieee80211n=1
wmm_enabled=1
ht_capab=[HT40][SHORT-GI-20][DSSS_CCK-40]
macaddr_acl=0
auth_algs=1
ignore_broadcast_ssid=0
wpa=2
wpa_key_mgmt=WPA-PSK
wpa_passphrase=LACONTRASENADEACCESO
rsn_pairwise=CCMP

    Here you can change some things:

    • The name of the network: this is configured in the line ssid= in our example, the network will be called SOLEVOLTAJE.
    • The channel: channel=11 is saying that we are going to operate on channel 11, but we can change it to whatever works best for our site.
    • The access password: for security, we need to assign a password to connect to the network. We do this by assigning it in the line wpa_passphrase=LACONTRASENADEACCESS.

    Now we must inform hostapd where the configuration is located. We do this by editing the :

    sudo nano /etc/default/hostapd

    … and adding in it the following line:

    DAEMON_CONF=

  • Moment 6: Start the services

    Before restarting our Pi, we must enable the services to run whenever we turn it on.

    To do this, type the following in the terminal:

    sudo systemctl unmask hostapd
sudo systemctl enable hostapd
sudo systemctl start hostapd

sudo systemctl unmask dnsmasq
sudo systemctl enable dnsmasq
sudo systemctl start dnsmasq

    … and finally, we restart the Pi to make our first test.

    sudo reboot now

  • Moment 7: Make a first test

    From another computer or phone, search in the wireless networks for the network we just created, in our case SOLEVOLTAJE and connect to it.

    If you check the network settings, you should see that the assigned IP address must be in the range we selected in the dnsmasq configuration . In our case, the device is assigned the IP 192.168.10.143.

  • Step 8: You can now share the wired connection

    At this point, those who connect to our access point will be able to see the files and services that we have configured on our Raspberry Pi. If you want to share the wired Internet connection through the wireless network continue with the following steps.

  • Step 9: Forward traffic to the wired connection

    It is necessary that our system forwards the traffic from the wireless interface to the wired one, in order to share the connection. At this point, we have to edit the file:

    sudo nano /etc/sysctl.conf

    In the contents of this, we look for the line that says #net.ipv4.ip_forward=1 and remove the # at the beginning to indicate that we want to activate this guideline. Once this modification is done, save the file and continue with the next step.

  • Step 10: Finally, secure the Firewall

    We are now in the last step of the configuration. Here we have to tell the Firewall to allow the transit of network traffic between the wireless and wired interfaces, for this, we enter these commands in our terminal:

    sudo iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
sudo iptables -A FORWARD -i eth0 -o wlan0 -m state --state RELATED,ESTABLISHED -j ACCEPT
sudo iptables -A FORWARD -i wlan0 -o eth0 -j ACCEPT

    At this point, we should already have internet on our wireless clients! However, we need to make sure that these changes are not lost when we reboot our Pi. To do this, we will save the configuration in a text file:

    sudo sh -c

  • Moment 11: Now, with Internet!

    After this long process, the clients connected to our Pi will be able to access the shared Internet connection from our Raspberry Pi… SUCCESSFUL!

Did you get tangled up in any of the steps? Tell us how to complete this solution to make it clearer.

How do you know if this solution works?

With this recipe, you can set up a portable Router, which is used to carry content on your [Raspberry Pi](https://sole-doc.tiddlyhost.com/#Raspberry Pi) and share it with the community, without the need for a constant Internet connection.

Also, if you have a wired connection, you can share it to make your SOLE sessions more powerful.

Why might this solution work?

Having a portable, low-power router allows us to easily bring our content to multiple locations where the Internet connection is not constant.

However, the number of clients our Pi can support is low. If you are going to connect more than 10 people at the same time, maybe the connection speed gets a little slow and, to solve this, you should look for a more powerful router.

If you have come this far, you have learned a lot of things that you can share with your community to improve connectivity conditions. Using the Internet in a group is a challenging experience, both in terms of technology and in terms of community. Doing SOLE is a way to use the Internet as a group to live better together.
